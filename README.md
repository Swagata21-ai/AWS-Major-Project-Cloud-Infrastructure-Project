# AWS Cloud Infrastructure Project 🚀

## Overview
This project demonstrates the design and deployment of a **Financial Tracker Application** on **Amazon Web Services (AWS)** using **Elastic Beanstalk**.  
It showcases secure, scalable, and highly available cloud architecture with automated load balancing, monitoring, and storage integration.

🔗 **Live Demo:** [Financial Tracker App](http://elasticapp.eba-chzm4pn4.us-east-1.elasticbeanstalk.com/)

---

## 🎯 Objectives
- Deploy a sample application using **AWS Elastic Beanstalk**.
- Implement secure architecture with **VPC (public & private subnets)**.
- Configure **IAM roles & policies** for access management.
- Enable **Auto Scaling & ELB** for traffic distribution.
- Integrate **CloudWatch** for monitoring and alerts.
- Use **S3** for static content storage and backups.

## 📐 Architecture
The project follows a **multi-tier architecture**:

1. **IAM** – Secure roles and permissions.
2. **VPC** – Public & private subnets, Internet Gateway, Route Tables, Security Groups.
3. **Elastic Beanstalk** – Application deployment with automated scaling.
4. **EC2 Instances** – Hosting application workloads.
5. **ELB & Auto Scaling** – Load distribution and traffic management.
6. **S3 Bucket** – Static content storage and backups.
7. **CloudWatch** – Monitoring, metrics, and alarms.

## ⚙️ Configurations
- **Step 1:** IAM setup (Service Role & EC2 Role for Elastic Beanstalk).
- **Step 2:** VPC creation with subnets, route tables, and security groups.
- **Step 3:** Elastic Beanstalk deployment of Financial Tracker App.
- **Step 4:** S3 bucket integration for static assets.
- **Step 5:** CloudWatch alarms and logging.
